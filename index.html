Capas de la Aplicación.

Para diseñar esta pequeña aplicacion se persiguió aplicar el patrón MVC (Modelo - Vista - Controlador), que separa el diseño en 3 capas: Presentación, Aplicación y Persistencia. La capa de Presentación es la que se encarga de la representación de la información, en este caso en un documento HTML (index.html), donde se tiene una especie de formulario para que el usuario ingrese los casos de prueba, con un botón que se encarga de enviar dicho string a ser procesado por la aplicación. Desde el botón "enviar" se llama a la clase controladora y se ejecuta el método enviar string (sendstring()) que recibe el string correspondiente al caso de prueba, necesario para realizar todas las operaciones y validaciones del Cube Summation. Aquí estaríamos en la capa de Aplicación, que es donde se ejecutan los procesos principales de la misma. Para finalizar, las clases de la capa de Aplicación llamarían a los métodos de la capa de Persistencia, la cual no existe en este ejercicio, ya que no se utilizó una Base de Datos. Los datos necesarios para ejecutar las funciones fueron obtenidos directamente desde el teclado.Sin embargo, esta capa se encarga de almacenar y recuperar datos de la Base de Datos.

Responsabilidad de clases.

Las clases del patrón MVC son las de Interfaz, clase Controladora y la case Entidad. La responsabilidad de la clase Interfaz es la de recibir las peticiones del usuario a la aplicación y la de mostrar el output. De igual manera, la responsabilidad de la clase controladora es la de gestionar o procesar la data dentro de la aplicación. Y la clase Entidad, la cual no fue utilizada en este ejercicio, se encarga de almacenar y gestionar datos de la BD.
